Mimic Me Mirror

An intelligent interactive system that recognizes human facial expressions using computer vision and displays corresponding expressions on an OLED screen while controlling servo-driven eyes.

January 2026
KLE TECHNOLOGICAL UNIVERSITY

Abstract

The Mimic Me Mirror is an innovative interactive system that integrates facial expression recognition with robotic actuation to create a responsive mirror interface. The system utilizes OpenCV Haar Cascade classifiers to detect facial expressions (Happy/Neutral) in real-time with 90-95% accuracy. Detected expressions are wirelessly transmitted via Bluetooth HC-05 module to an Arduino Nano microcontroller, which controls dual servo motors for eye movement and displays the recognized expression on a 128x64 OLED display.

The project successfully demonstrates the integration of computer vision, wireless communication, embedded systems, and mechanical control. Key achievements include real-time response latency of 150-200ms, bandwidth-optimized data transmission (sending data only on expression changes), and a cost-effective implementation (₹3,400). The system provides dual feedback through both visual (OLED display) and mechanical (servo eyes) outputs, creating an engaging interactive experience with potential applications in educational demonstrations, interactive installations, and accessibility interfaces.

Project Description & Workflow

1

Video Capture

USB camera captures video frames at 30 FPS with 640x480 resolution. The system works best at 0.5-2m distance with controlled lighting (300-500 lux).

2

Facial Expression Detection

Python script using OpenCV processes frames, detects faces with Haar Cascade classifiers, and identifies smiles. Accuracy: 90-95% under optimal conditions.

3

Data Encoding & Transmission

Detected expression encoded as 'H' (Happy) or 'N' (Neutral). Data sent via HC-05 Bluetooth only when expression changes, reducing bandwidth by 80%.

4

Arduino Processing

Arduino Nano receives data on pins 2-3 via SoftwareSerial, processes the expression code, and prepares output commands for servos and OLED.

5

Dual Output Activation

Servo motors move eyes (60°/120° for Happy, 90°/90° for Neutral) while OLED displays corresponding expression graphics with ~60 FPS refresh rate.

Challenges & Solutions

Poor Smile Detection in Side Profile

Haar Cascade classifiers performed poorly when faces were not frontal, especially in detecting smiles from side angles.

Enforce Frontal Face Constraint

Implemented facial angle detection and only process frames where face is within ±30° of frontal view. Improved lighting to 300+ lux for better detection.

OLED Display Flicker

OLED screen exhibited flickering and unstable graphics during operation, especially when servos were moving.

Stable Power Supply & Correct I2C Wiring

Ensured stable 5V supply with proper filtering capacitors. Verified correct I2C wiring (SDA/SCL pins) and added pull-up resistors for stable communication.

Servo Jitter Under Load

Servo motors exhibited jittery movement and inconsistent positioning when both servos and OLED were active simultaneously.

External Power Supply with Proper Grounding

Used 5V 2A external power supply instead of USB power. Implemented proper common ground connections between all components to eliminate ground loops.

Bluetooth Connection Drops

Intermittent Bluetooth disconnections and data packet loss, especially when operating near other wireless devices.

Voltage Divider Configuration

Verified and corrected voltage divider circuit (2.2kΩ + 1kΩ) on RX line to ensure proper 3.3V logic levels for HC-05 module communication.
